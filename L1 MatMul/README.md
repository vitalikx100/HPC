# Лабораторная работа №1 

**Перемножение матриц**

**Задача**: реализовать алгоритм перемножения матриц

**Язык**: C++ или **Python**

**Входные данные**: 2 матрицы размером от 100х100 до 2000х2000 каждая.

**Выходные данные**: проверка корректности перемножения + время вычисления

Реализация должна содержать 2 функции перемножения матриц: на CPU и на GPU с применением CUDA.

# Реализация
Функция **multiply_cpu** реализует умножение матриц на CPU: вызов функции **np.dot()** библиотеки NumPy, которая реализует матричное умножение. Функция возвращает итоговую матрицу произведения и время выполнения.

Функция **multiply_gpu** реализует умножение матриц на GPU с использованием **CUDA**: c помощью функции **cuda.mem_alloc()** выделяется память на GPU для хранения матриц, функция **cuda.memcpy_htod()** копирует данные из оперативной памяти на GPU. 

Используется блок размером **32х32** потока, размер сетки расчитывается для покрытия всей матрицы. Для измерения времени создаются два объекта cuda.Event() для фиксации времени до и после выполнения.

Запускается CUDA ядро, где каждый поток вычисляет один элемент итоговой матрицы. Результат копируется в оперативную память с помощью функции **cuda.memcpy_dtoh()**. Функция возвращает итоговую матрицу произведения и время выполнения.

Полученные значения итоговых матриц функций **multiply_cpu** и **multiply_gpu** сравниваются с помощью функции **np.allclose()**, выполняющей поэлементную проверку равенства с учетом допустимой погрешности.

Для матрицы размерностью 100х100 время выполнения на GPU **превышает** время на CPU, что объясняется накладными расходами на запуск ядра и малым объемом вычислений. Начиная с размерности матриц 200х200 GPU начинает демонстрировать преимущество перед вычислением на CPU. Рост времени выполнения на GPU происходит значительно медленнее по сравнению с CPU, что связано с высокой степенью параллелизма вычислений.
