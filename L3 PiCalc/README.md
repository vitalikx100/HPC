# Лабораторная работа №3

**Вычисление π**

**Задача**: реализовать алгоритм вычисления π методом Монте-Карло с использованием CUDA

**Язык**: C++ или **Python**

**Входные данные**: N - количество точек

**Выходные данные**: значения π, время выполнения на GPU и CPU

# Реализация
Функция **compute_pi_cpu** реализует вычисление **π** на **CPU**: генерируются два массива случайных координат с помощью **np.random.rand** и далее вычисляется количество точек, попавших внутрь четверти окружности (х^2 + y^2 <= 1). Результат умножается на 4/N, где N - количество точек, для получения значения **π**.

Функция **compute_pi_gpu** реализует вычисление **π** на **GPU** с использованием **CUDA**: c помощью генератора **curandom.XORWOWRandomNumberGenerator** генерируется массив случайных чисел в видеопамяти GPU, функция **cuda.mem_alloc** выделяет память на GPU для переменной счетчика, а **cuda.memcpy_htod** инициализирует ее нулевым значением.

Используется блок размером **256** потоков, размер сетки рассчитывается по формуле **math.ceil(N / block_size)** для покрытия всего массива точек.

Происходит запуск CUDA ядра, где каждый поток обрабатывает одну точку (пару координат): получает координаты и проверяет попадание в четверть окружности (х^2 + y^2 <= 1) и записывает значение 1 в **shared memory** в случае успешного попадания. Далее выполняется редукция, где происходит попарное суцммирование внутри блока с использованием разделяемой памяти. После редукции один поток из каждого блока атомарно добавляет полученную сумму к глобальному счетчику с помощью **atomicAdd**.

Полученный результат копируется в оперативную память с помощью **cuda.memcpy_dtoh** и далее вычисляется значение **π** по формуле, аналогичной в **compute_pi_cpu**.

С увеличением числа точек время выполнения на CPU растет линейно, так как каждая точка обрабатывается последовательно: от 182 мс при 10.000.000 до 9227 мс при 500.000.000 точек.

<img width="633" height="474" alt="{645C4EED-05C0-4195-8909-45348E32D313}" src="https://github.com/user-attachments/assets/0db46dcb-3476-463d-860e-a5c896f274f2" />

Время выполнения на GPU находится в диапазоне от 2436 мс до 2934 мс при количестве точек от 10.000.000 до 500.000.000. Существенное влияние оказывают накладные расходы, связанные с генерацией случайных чисел, управлением памятью и запуском ядра. С увеличением N незначительно растет время выполнения, в связи с высокой степенью параллелизма вычислений.

<img width="632" height="472" alt="{C9ED6435-1C25-4596-822E-4BDCD08E34F4}" src="https://github.com/user-attachments/assets/d8453e95-0f16-4370-8c66-54252a16fdfa" />

Для объемов данных до 100.000.000 GPU уступает CPU (ускорение меньше 1) в связи с накладными расходами и относительно малым объемом вычислений. Эффективность вычислений на GPU начинается преобладать при 200.000.000 точек (ускорение в 1,38 раз). При дальнейшем увеличении N преимущество вычислений на GPU усиливается (ускорение возрастает до 3,15), демонстрируя преимущество вычислений на GPU для больших объемов данных.
