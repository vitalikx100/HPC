# Лабораторная работа №2 

**Сумма элементов вектора**

**Задача**: реализовать алгоритм сложения элементов вектора

**Язык**: C++ или **Python**

**Входные данные**: Вектор размером 1 000..1 000 000 значений.

**Выходные данные**: сумма элементов вектора + время вычисления

Реализация должна содержать 2 функции сложения элементов вектора: на CPU и на GPU с применением CUDA.

# Реализация
Функция **sum_cpu** реализует суммирование элементов вектора на **CPU**: вызов функции **np.sum()** библиотеки NumPy, которая вычисляет сумму всех элементов вектора. Функция возвращает сумму элементов и время выполнения в миллисекундах.

Функция **sum_gpu** реализует суммирование элементов вектора на **GPU** с использованием **CUDA**: с помощью функции **cuda.mem_alloc()** выделяется память на GPU для хранения вектора и итоговой переменной. Функция **cuda.memcpy_htod()** копирует исходный вектор из оперативной памяти на GPU, а также переменную для результата.

Используется блок размером **256** потоков, размер сетки рассчитывается по формуле **math.ceil(N / block_size)** для покрытия всего вектора. Для измерения времени создаются два объекта **cuda.Event()**, которые фиксируют временные метки до и после выполнения ядра.

Запускается CUDA ядро, реализующее **алгоритм редукции** с использованием **shared memory**. Каждый поток загружает свой элемент в память блока, после чего выполняется **попарное суммирование** внутри блока. Затем один поток из каждого блока **атомарно добавляет полученную сумму** к глобальному результату с помощью **atomicAdd()**. Это позволяет значительно сократить количество операций к глобальной памяти.

После завершения вычислений результат копируется обратно в оперативную память с помощью функции **cuda.memcpy_dtoh()**. Функция возвращает сумму элементов вектора и время выполнения на GPU.

Полученные значения суммы, вычисленные функциями sum_cpu и sum_gpu, сравниваются с помощью функции **np.allclose()**, выполняющей сравнение с учетом допустимой погрешности для проверки корректности вычислений.

Для малых размерностей (1000-10.000 элементов) время выполнения на CPU остается на уровне 0,01 мс, что связано с малым объемом вычислений. При увеличении до 1.000.000 элементов время увеличивается до 0,65 мс, что демонстрирует линейную зависимость.

<img width="632" height="472" alt="{E0A1E34B-4592-4D55-BE0A-B9562584E548}" src="https://github.com/user-attachments/assets/3476d75d-f4cb-41bf-b645-a63f36806c67" />

Время выполнения на GPU для малых размерностей вектора определяется накладными расходами на запуск ядра и передачу данных. С увеличением размерности время сохраняется в диапазоне 0,04-0,05 мс, что указывает на эффективную параллельную обработку.

<img width="629" height="472" alt="{C1EC94EE-C993-4079-B03B-8492D565B336}" src="https://github.com/user-attachments/assets/11845015-608f-4377-91a3-94424f83864e" />

Для малых размерностей вектора ускорение меньше единицы, что объясняется преобладанием накладных расходов GPU в сравнении с вычислительной нагрузкой. Начиная приблизительно с размерности в 200.000 элементов использование вычислений на GPU становится целесообразным (ускорение 1,54). Максимальное ускорение составило 15,72 при размерности в 1.000.000 элементов, демонстрируя эффективности расчета на GPU на большем объеме данных.
